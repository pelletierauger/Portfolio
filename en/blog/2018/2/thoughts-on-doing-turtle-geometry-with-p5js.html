Trying out Turtle geometry with p5.js

How could we apply the ideas of Seymour Papert inside p5.js?
February 15, 2018

In the summer of 2017, I read two wonderful books by Seymour Papert that profoundly changed my way of thinking about learning and education: Mindstorms and The Children’s Machine. I read Mindstorms as part of the p5.diversity interactive book club. It was an amazing experience. Papert was a passionate educator, mathematician, and computer scientist who believed that computers could and should change the way we learn. One of his most interesting inventions is Turtle geometry, a tool to learn mathematics in an embodied and intuitive way. While reading about it, I couldn’t help wanting to try it for myself. I kept wondering what would be the best way to do Turtle geometry today. This article is about my first foray into it, using p5.js.

What is Turtle geometry?

There are a lot of things to say about the fascinating theory and immense potentiel of Turtle geometry, but for now I will just describe it as simply as possible for the purposes of this embryonic project.

In 1967, Seymour Papert created LOGO, a computer system and software toolkit to build educational experiences. One of the key part of LOGO is the Turtle, a mathematical object that only has two properties: a position in space and a heading (a direction in which its “head” is pointing). To make the Turtle move, you give it instructions in a language called TURTLE TALK. For example, the command FORWARD 50 makes the turtle go forward 50 steps. The command LEFT 180 makes it rotate left by 180 degrees. The code below makes the Turtle draw a square: 

FORWARD 100
RIGHT 90
FORWARD 100
RIGHT 90
FORWARD 100
RIGHT 90
FORWARD 100
RIGHT 90

To make a series of instructions (a program) easier to read and write, LOGO also has a REPEAT command. It repeats every commands before the word END, the number of times indicated to the right of the command REPEAT. This is another way to draw the same square: 

REPEAT 4
    FORWARD 100
    RIGHT 90
END

The Turtle has other interesting commands, for example PENDOWN lowers a pen to draw the path of the Turtle, and PENUP lifts it. Yes, this does mean that a Turtle has more properties than just position and heading. While the mathematical theory behind Turtle geometry requires only position and heading, you’ll see soon that playing with Turtles usually involves giving them more properties.You can also create custom functions, with this syntax: 

TO SQUARE
    REPEAT 4
        FORWARD 100
        RIGHT 90
    END
END

To continue the analogy of talking to a Turtle, creating custom functions was something Papert explained as “teaching the Turtle a new word”. Your custom functions can also have arguments, using this syntax: 

TO SQUARE WIDTH
    REPEAT 4
        FORWARD WIDTH
        RIGHT 90
    END
END

You can then call SQUARE 500 and get a square with a width of 500. These are the basic building blocks. This is very simplified, but we can now move on to the question of replicating this with p5.js.

Replicating TURTLE TALK with p5.js

On the surface, it might seem like a trivial problem to replicate TURTLE TALK, but I think it is deceptively intricate. Let’s say we create a Turtle class, and then we give it methods like forward(), left(), and right(). Assuming that the methods only do what they are supposed to (move and rotate the Turtle), we would write the following code to create a new Turtle and make it draw a square: 

function draw() {
    let t = new Turtle();
    t.forward(100);
    t.right(90);
    t.forward(100);
    t.right(90);
    t.forward(100);
    t.right(90);
    t.forward(100);
    t.right(90);
}

The unfortunate thing here is that we will not get to see the Turtle move. The square will appear fully formed, because all the Turtle methods are called on the same frame of animation.

This is unfortunate because Turtle geometry is meant to be a way to learn mathematics using our senses and our intuitions, creating mental links between abstractions and experiences. A film shot at mit in 1983 shows a group of children doing Turtle geometry with LOGO, and we can see the Turtles moving on the computer screens. LOGO did not only display the results of a series of commands but the process too. The film also shows a mechanical Turtle: a small robot that takes TURTLE TALK commands and moves over a piece of paper to draw the resulting shapes. While users of LOGO would observe the robot as it moved and drew, the JavaScript code I wrote above is more like entering a room where a robot has already finished drawing something. It builds an incomplete sensory bridge between the abstract Turtle commands and their visual, physical results.

So how could we go about replicating TURTLE TALK’s friendly syntax and also get an animated Turtle as a result? This would be a simple way to achieve it: 

function draw() {
    let t = new Turtle();
    let instructions = [
        t.forward(100),
        t.right(90),
        t.forward(100),
        t.right(90),
        t.forward(100),
        t.right(90),
        t.forward(100),
        t.right(90)
    ];
    if (instructions[frameCount - 1]) {
        instructions[frameCount - 1]();
    }
}

The problem here is that we cannot use the REPEAT syntax or easily create our own functions like SQUARE. It’s possible to do it, but it’s very inconvenient.

A way to make it work

With all of that in mind, I created a simple (but somewhat strange) system that addresses most issues. I certainly do not claim that it is the way to replicate TURTLE TALK in p5.js. I’m just experimenting and testing out ideas. This is how I create a Turtle that draws a square: 

let t = new Turtle({
    instructions: function() {
        forward(100);
        right(90);
        forward(100);
        right(90);
        forward(100);
        right(90);
        forward(100);
        right(90);
    }
});

Two things are strange here. Firstly, the forward(100) and right(90) calls are not methods of the Turtle class, they are public functions. Secondly, this is not visible here but those public functions do not actually do what they “claim” to be doing. They are instead pushing objects in a global array called instructionBox: 

function forward(size) {
    instructionBox.push({
        f: "forward",
        s: size
    });
}

function right(angle) {
    instructionBox.push({
        f: "right",
        a: angle
    });
}

These objects are instructions to be executed later. Once a series of them are inside the instructionBox array, the system is able to execute them, one per animation frame, letting us see the Turtle’s process.

In the Turtle class below, you can see that when an instance of Turtle is created, it calls the instructions() function that was sent to it, and then collects the results contained in the global instructionBox array, saving it in a private property called states: 

var Turtle = function(t) {
    this.heading = t.heading || 0;
    this.position = t.position || new p5.Vector(0, 0);
    this.penDown = true;
    this.awake = (t.awake != null) ? t.awake : true;
    if (this.awake) {
        // We call the instructions() function, filling instructionBox.
        t.instructions();
        // We collect the resulting instructions in this.states.
        this.states = instructionBox;
        // And we empty instructionBox to make room for new turtles.
        instructionBox = [];
        turtles.push(this);
    }
};

The constructor() also lets you see all the properties defining an instance of Turtle:

  • —  position and heading are most basic. You can see that position has a default value of (0, 0) and heading has a default value of 0.
  • —  penDown is a boolean letting us know if the Turtle’s pen is down or up. It defaults to true.
  • —  awake lets us create many turtles and not display all of them. I found out while doing my first experiments with Turtle geometry that it is very useful to create a lot of Turtles with different sets of instructions, and commenting out Turtles to better see others is a very poorly designed experience. So this awake boolean, which defaults to true, lets us make some turtles “sleep”.

The fact that forward(100) and right(90) are public functions allows us to use them inside repeat() functions at will, without ever worrying about the scope. This would not work (I actually need to test it): 

var marguerite = new Turtle({
    awake: false,
    heading: 0,
    position: new p5.Vector(10, 0),
    instructions: function(t) {
        var s = 10;
        repeat(3, function() {
            repeat(30, function() {
                t.forward(s);
                t.right(20);
                s += 0.1;
            });
            repeat(60, function() {
                t.forward(s);
                t.left(20);
                s += 0.1;
            });
        });
    }
});

I always thought that JavaScript should have a simple repeat syntax. I first learned programming with HyperCard and thankfully it had one. I think a simple repeat syntax is a great learning tool for beginners. It’s the friendliest approach possible to loops. Also, how often do we use a for loop just to repeat an action a fixed amount of times? The following syntax could be very useful: 

// This is not correct JavaScript but maybe it should be:
repeat 4 {
    console.log("This is such a gentle introduction to looping!");
}

Is there some underlying reasons why this couldn’t be done? Would it introduce any problem that I’m unaware of? Or is it just dumb? Here is the code to my very simple proposal for a repeat() function that offers similar functionality: 

// The repeat() function takes 2 arguments: 
// n is the amount of times it should loop.
// f is the function to call n amount of times.
function repeat(n, f) {
    for (var i = 0; i < n; i++) {
        // We send the index i to the function, for optional use.
        f(i);
    }
}

So now, our square can be drawn with this bit of code, which is quite close to its LOGO equivalent: 

repeat(4, function() {
    forward(100);
    right(90);
});

The repeat() function can also be nested at will: 

repeat(4, function() {
    forward(100);
    right(90);
    // Let's draw a smaller square at each corner of the big square.
    repeat(4, function() {
        forward(10);
        right(90);
    });
});
var marguerite = new Turtle({
    awake: false,
    heading: 0,
    position: new p5.Vector(10, 0),
    instructions: function() {
        var s = 10;
        repeat(3, function() {
            repeat(30, function() {
                forward(s);
                right(20);
                s += 0.1;
            });
            repeat(60, function() {
                forward(s);
                left(20);
                s += 0.1;
            });
        });
    }
});
